# 公众号排版规范

公众号最终在手机上、富文本编辑器里呈现。这里只管**写作阶段的 Markdown**——发布时的样式（字号、颜色、行距）由编辑器或排版工具处理，不在本 skill 范围。但下面这些写进 Markdown 就能少返工。

## 中英文混排

- 中文与英文 / 数字之间加**一个半角空格**：`用 GPT-4 写` 不是 `用GPT-4写`、`大概 70%` 不是 `大概70%`
- 中文标点（，。！？：；）不加空格，本身已含间距
- 行内代码、变量、命令、文件名用反引号：`account.move`、`npm install`

## 标点

- 正文用**全角中文标点**：，。！？「」（）—— 不用半角 `,.!?`
- 引号用「」『』或中文双引号；英文术语内部保留原样
- 省略号用 `……`（两个），不用 `...`
- 破折号用 `——`（两个全角），不用 `--` 或 `-`

## Markdown 语法

- 标题从 `##` 起，不跳级（正文标题已在 frontmatter，body 不再放 H1，避免重复）
- 加粗用 `**金句**`——只加粗会被截图转发的那 2-4 句，不整段加粗
- 小标题具体、带信息：`3 天搞定 vs 传统方案` 胜过 `方案对比`
- 并列项拆成 `- 列表`；步骤用 `1. 有序列表`；对比用表格
- 金句、警告、引文用 `> 引用`

## 段落节奏（手机视角）

- **一段一个意思**，3-4 行封顶——长段在手机上是一堵看了就想划走的墙
- 长句拆短句。一句话塞三个转折，读者在小屏上跟不动
- 段与段之间留白，别堆成砖块
- 重要的话单独成段，给它呼吸

## 配图位

写作时在该配图的地方留一个占位说明，交给下游 skill 出图：

```
> [配图：封面 — 一句话点题的视觉，方向 XXX]
> [配图：文中 1 — 小黑 scene=funnel，label=筛选；node scripts/xiaohei-svg.mjs funnel --label 筛选 --png]
```

正文配图默认走 **小黑 IP**（黑色实心小怪物，承担概念的动作主体），用 SVG 脚本画、不靠图像模型——挑 scene + `--label`，见 [xiaohei-ip.md](xiaohei-ip.md)。封面/模板类视觉仍可交 `alanhou-social-card`（公众号 21:9 + 1:1 对）与 `alanhou-card`（文中长图/卡片）。
