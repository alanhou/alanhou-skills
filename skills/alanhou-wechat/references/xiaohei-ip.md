# 小黑 · 正文配图固定视觉 IP

小黑是中文正文配图的**固定视觉 IP**。本 skill 默认每张正文配图都让小黑出场——它不是吉祥物、不是贴纸、不是可爱装饰，而是**正在认真参与系统运转的荒诞工作者**。

**小黑用 SVG 画，不靠图像模型**：`scripts/xiaohei-svg.mjs` 用代码渲染，确定、可编辑、无 API key。

## 怎么用

```bash
node scripts/xiaohei-svg.mjs <scene> [--label 文字] [--png] [-o out.svg]
```

- 默认输出 `~/Downloads/xiaohei-<scene>.svg`，`-o` 指定路径
- `--label` 改场景里的文字（如拉杆/印章/箱子上的字）
- `--png` 顺便渲一张 PNG（用已装的 Playwright）。**公众号不收 SVG，要 `--png`，再把 PNG 传进编辑器素材库**（防盗链，外链图发布后不显示）
- SVG 可直接改：颜色、文字、路径都在一个文件里

## 场景目录

每个场景都让小黑**承担那个概念的物理动作**：

| scene | 表达的概念 | 小黑在做什么 | 默认 label |
|-------|-----------|-------------|-----------|
| `idle` | 通用/出场 | 站着，呆呆看你 | — |
| `funnel` | 信息过载、要筛选 | 本体变成漏斗，杂乱倒进去只漏一两条 | — |
| `breakpoint` | 流程卡在某环节 | 卡在断开的线里，伸手够不到对岸 | 卡住了 |
| `lever` | 判断/决策 | 认真扳一根写着「判断」的拉杆 | 判断 |
| `stamp` | 承接/转化话术 | 举印章往飘过来的一句话上盖 | 承接 |
| `warning` | 风险预警 | 举「小心」牌站在大坑边往里看 | 小心 |
| `carry` | 搬运素材/资源 | 怀里抱着一个「素材」箱 | 素材 |

写配图时：先找出这张图要表达的文章概念，挑最接近的 scene，用 `--label` 换成文章里的词。

## 外形（画法规范）

- 黑色实心小怪物，白色圆点眼睛
- 细腿，偶尔细胳膊；轮廓略不规则、有手绘感（靠 SVG 的 `feTurbulence` 位移滤镜抖一下）
- 身体可以是圆柱、黑豆、黑盒、漏斗、影子、洞口、机器内部黑块
- 表情空、呆、冷静、认真
- 背景：极简白板草图（off-white + 浅网格），单色为主，最多一点马克笔强调色

## 性格

很认真但做的事有点荒诞；像低调的系统操作员；冷幽默不卖萌；有点笨拙但不蠢；像在白板草图里真的负责某个工作。

## 禁止

- 不要过度可爱的吉祥物 / 儿童卡通角色
- 不要复杂服装、表情包、闪亮眼睛
- 不要让小黑只站在角落里看
- 不要让小黑抢走结构表达
- 不要太商业、太圆润、太精致
- **黑物件别画在黑身上**（看不见）——要小黑「拿着」的东西，用描边留白（白底黑线），别用实心黑

## 判断标准（每张图必过）

**去掉小黑，图的核心隐喻还完全成立吗？**

- 成立 → 小黑太装饰了，换个让小黑成为动作主体的 scene，或加新 scene
- 不成立（小黑就是那个动作本身）→ 通过

## 加新场景

`scripts/xiaohei-svg.mjs` 里 `scenes` 对象每个键是一个场景，返回一段 SVG。复用 `body({arms})` 画小黑本体，自己画道具，让道具落在小黑手/身的延长线上（参考 `lever`/`stamp`：小黑在左，道具在右，用一条显式手臂连过去）。新场景同样要过上面的判断标准。
