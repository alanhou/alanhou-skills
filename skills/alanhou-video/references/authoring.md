# 场景 HTML 写作规则

每个场景是一个独立的单文件 HTML，渲染器（`scripts/render.mjs`）在无头 Chromium 里录制它。理解渲染器的行为，场景才不会被截断或拖长。

## 时长探测（最重要）

渲染器在 `-d auto`（默认）下这样决定录多久：

- 扫描所有元素的 CSS 动画，取**非 infinite** 动画的最大 `duration + delay`
- 如果页面有 GSAP，取**非 repeat:-1** tween 的最长 totalDuration
- 两者取大，加 0.4 秒收尾余量，上限 30 秒；都没有则默认 5 秒

由此得出的写法规则：

1. **内容动画（入场、画线、打字）必须是有限次**——这是时长的依据
2. **背景氛围动画（漂浮、闪烁、扫描线）写成 `infinite`**——探测会忽略它们，不会把片段拖到 30 秒
3. 想精确控制每场景时长，用 `-d <秒>`：动画提前结束就定格最后一帧补齐
4. GSAP 主时间线想被完整录制：注册成 paused 的 `window.__timelines["main"] = tl`，渲染器会在 t=0 调 `play(0)`

## 冻结与 t=0

渲染器在文档解析前冻结所有 CSS 动画，等字体加载完才解冻——解冻那一刻是动画的 t=0，前面的死时间会被 ffmpeg 剪掉。所以：

- 不要用 `setTimeout` 在 load 后才挂关键动画（冻结对它无效，时机会错）
- 用纯 CSS `animation: ... forwards` 或 paused GSAP timeline，让渲染器掌握时钟

## 字体与外部资源

- Google Fonts 可用，渲染器会等全部字面下载完（上限 8 秒）再开录，不会出现字体闪换
- Tailwind CDN 可用；其他外部 JS 尽量内联，跨域视频/大图会拖慢加载
- 图片用本地相对路径或 data URI 最稳

## 版式

- 默认画布 1920x1080，`body { overflow:hidden }`，所有内容绝对定位或 flex 居中
- 安全区：四边留 5% 以上，电视/播放器可能裁边
- 文字最小 24px（1080p 下再小就糊）；中文标题用系统字重兜底 `"Noto Sans SC", "PingFang SC"`
- 竖屏（1080x1920）不要直接复用横屏场景——重排版式再渲染，`-s 1080x1920`

## 多场景节奏

- 片头（钩子）3-5 秒 → 内容场景每个 4-8 秒 → outro 3 秒
- 相邻场景换风格要克制：同一个片子最多 2 种视觉家族（如 glitch 开场 + cinema 正文 + logo 收尾可以；六种模板各来一遍不行）
- 拼接是硬切。需要转场就把转场做进场景自身的开头/结尾动画（淡入淡出、扫光）
